# Internacionalização Satriz (PT · EN · ES) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Site público em 3 idiomas com URLs indexáveis (`/` PT, `/en`, `/es`), auto-detecção de idioma, seletor na nav e hreflang — admin e APIs intocados.

**Architecture:** `next-intl` com segmento `app/(site)/[locale]/` (route group) pras páginas públicas e `app/(admin)/` pro admin (dois root layouts). Middleware único: auth do admin primeiro, i18n depois. Todo texto visível migra para `messages/{pt,en,es}.json`.

**Tech Stack:** Next.js 16 (App Router), next-intl, Tailwind v4, TypeScript.

**Spec:** `docs/superpowers/specs/2026-07-03-i18n-design.md`

## Global Constraints

- Locales: `pt` (default, SEM prefixo na URL — `localePrefix: "as-needed"`), `en`, `es`.
- Slugs iguais nos 3 idiomas (`/en/servicos`, NÃO `/en/services`).
- `/admin`, `/api/*` e o painel de leads: comportamento byte-a-byte igual ao atual (PT, protegidos).
- Componentes NÃO usados (Club, ClubMini, Estatuto, Divisoes, Contato, ScrollVideo) ficam intocados — sem tradução.
- Sem framework de teste no projeto: cada task verifica com `npx tsc --noEmit` + `curl` no dev server (localhost:3000) com greps do conteúdo esperado.
- O dev server precisa de restart manual após mudanças em `next.config.ts` e `middleware.ts` (avisar o usuário; ele roda o server).
- Commit ao fim de CADA task (repo já inicializado, baseline `5402501`).
- Nomes de marca nunca se traduzem: "Satriz Club", "Satriz Studio", "Satriz Revista", "Satriz News", "Software House".

**Interfaces:**

- [ ] **Step 1: Instalar next-intl**

Run: `cd /d/PROJETOS/Satriz && npm install next-intl`
Expected: adiciona `next-intl` em dependencies, sem erros.

- [ ] **Step 2: Criar `i18n/routing.ts`**

```ts
import { defineRouting } from "next-intl/routing";

export const routing = defineRouting({
  locales: ["pt", "en", "es"],
  defaultLocale: "pt",
  // PT fica sem prefixo (URLs atuais preservadas); /en e /es prefixados.
  localePrefix: "as-needed",
});
```

- [ ] **Step 3: Criar `i18n/request.ts`**

```ts
import { getRequestConfig } from "next-intl/server";
import { hasLocale } from "next-intl";
import { routing } from "./routing";

export default getRequestConfig(async ({ requestLocale }) => {
  const requested = await requestLocale;
  const locale = hasLocale(routing.locales, requested) ? requested : routing.defaultLocale;
  return {
    locale,
    messages: (await import(`../messages/${locale}.json`)).default,
  };
});
```

- [ ] **Step 4: Criar `i18n/navigation.ts`**

```ts
import { createNavigation } from "next-intl/navigation";
import { routing } from "./routing";

// Navegação locale-aware: mantém o idioma atual nos hrefs e permite trocar
// de locale via router.replace(pathname, { locale }).
export const { Link, redirect, usePathname, useRouter, getPathname } =
  createNavigation(routing);
```

- [ ] **Step 5: Plugar o plugin no `next.config.ts`**

Conteúdo atual:
```ts
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  /* config options here */
};

export default nextConfig;
```

Novo conteúdo:
```ts
import type { NextConfig } from "next";
import createNextIntlPlugin from "next-intl/plugin";

const withNextIntl = createNextIntlPlugin("./i18n/request.ts");

const nextConfig: NextConfig = {
  /* config options here */
};

export default withNextIntl(nextConfig);
```

- [ ] **Step 6: Verificar tipos**

Run: `cd /d/PROJETOS/Satriz && npx tsc --noEmit`
Expected: exit 0 (messages/*.json ainda não existe, mas request.ts usa import dinâmico — não quebra o tsc).

- [ ] **Step 7: Commit**

```bash
git add package.json package-lock.json i18n next.config.ts
git commit -m "feat(i18n): instala next-intl e cria configuração de routing/request/navigation"
```

- [ ] **Step 6: Reescrever `middleware.ts`** (auth primeiro, i18n depois)

```ts
import { NextResponse, type NextRequest } from "next/server";
import createMiddleware from "next-intl/middleware";
import { routing } from "./i18n/routing";
import { ADMIN_COOKIE, isAuthed } from "@/lib/auth";

const intl = createMiddleware(routing);

export function middleware(req: NextRequest) {
  const { pathname } = req.nextUrl;
  const authed = isAuthed(req.cookies.get(ADMIN_COOKIE)?.value);

  // 1) Auth do admin — comportamento idêntico ao anterior, SEM locale.
  if (pathname.startsWith("/api/leads")) {
    if (!authed) return NextResponse.json({ error: "Não autorizado" }, { status: 401 });
    return NextResponse.next();
  }
  if (pathname.startsWith("/admin") && !pathname.startsWith("/admin/login")) {
    if (!authed) {
      const url = req.nextUrl.clone();
      url.pathname = "/admin/login";
      return NextResponse.redirect(url);
    }
    return NextResponse.next();
  }
  if (pathname.startsWith("/admin") || pathname.startsWith("/api")) {
    return NextResponse.next();
  }

  // 2) i18n — detecção Accept-Language + cookie NEXT_LOCALE + redirects.
  return intl(req);
}

export const config = {
  // Tudo exceto assets do Next e arquivos estáticos (com extensão).
  matcher: ["/((?!_next|.*\\..*).*)"],
};
```

const navegar = useCallback(
  (href: string, locale?: string) => {
    const alvo = locale ?? localeAtual;
    // mesmo caminho E mesmo idioma → nada a fazer
    if (href === pathname && alvo === localeAtual) return;
    destino.current = { href, locale: alvo };
    setFase("cobrir");
  },
  [pathname, localeAtual]
);
```

useEffect(() => {

    fase === "coberto" &&
    destino.current &&
    pathname === destino.current.href &&
    localeAtual === destino.current.locale
  ) {
    destino.current = null;
    const id = requestAnimationFrame(() => requestAnimationFrame(() => setFase("revelar")));

// hreflang alternates pra uma rota do site (slugs iguais nos 3 idiomas).
export function altFor(path: string, locale: string) {
  const clean = path === "/" ? "" : path;
  return {
    canonical: locale === "pt" ? `${clean || "/"}` : `/${locale}${clean}`,
    languages: {
      "pt-BR": clean || "/",
      en: `/en${clean}`,
      es: `/es${clean}`,
      "x-default": clean || "/",
    },
  };
}
```
